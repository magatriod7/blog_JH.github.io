기존의 웹 서비스는 a태그를 이용해서 해당 페이지로 이동하게 되면
새로운 웹 페이지를 브라우저에 나타내는 방식이었습니다.

사진 1

렌더링이 빠르다는 장점이 있었지만 새로운 페이지로 옮길 때마다
매번 모든 데이터를 받아야한다는 단점이 있었습니다.

그러던 중 Ajax를 이용한 일부분의 데이터만 받아온 후 해당 부분의 데이터를 변경하는
방식이 사용되었습니다.

사진 2

필요한 데이터만 새로 그리기 때문에 낭비가 없고 클라이언트한테도 친화적인
방벙이었지만 URL이 바뀌지 않아 히스토리 관리가 어렵고 JS 코드가 너무 복잡해지는 단점이 있었습니다.

사진 3

히스토리 관리가 어려운 Ajax 방식을 보완하고자 나타난 것이 SPA 방식입니다.

SPA 방식은 부분적인 렌더링을 함으로써 데이터에 낭비가 없고 히스토리가 남아
웹페이지를 관리하기 더 쉬워집니다. 직접 구현하기에는 보다 복잡해진
단점도 있지만 현재 react나 vue 등 많은 라이브러리가 생겨 보다 쉽게
웹페이지를 관리할 수 있게 되었습니다.

처음 SPA 라우팅 방식으로 만들어 졌던 것은 hash를 이용한 것이었습니다.
test.com/#2와 같은 해시 방식은 history를 남기는 점에서 Ajax 방식을 해결했지만
불필요한 #이 들어가게 됩니다. 또 다른 이슈로 현재 웹 상에서 많이 쓰이는
웹 크롤러가 해시 방식으로 작동할 경우 사이트의 콘텐츠를
수집할 수가 없게 된다는 점이 있습니다.

현재 리액트에서는 Link 태그를 사용하고 있습니다. Link 태그는 preventDefault()를
호출하여 a태그로 html 상에서 나타난 Link 태그를 멈추고 URL 정보에 따라 렌더링을 한후에
history.push()를 이용하여 주소창의 주소를 바꾸게 됩니다.
